# Contributing to AI QA Evaluation Lab

First off, thanks for taking the time to contribute! 🎉

The following is a set of guidelines for contributing to AI QA Evaluation Lab. These are mostly guidelines, not rules. Use your best judgment, and feel free to propose changes to this document in a pull request.

## How Can I Contribute?

### Reporting Bugs
This section guides you through submitting a bug report. Following these guidelines helps maintainers and the community understand your report, reproduce the behavior, and find related reports.

- **Check if the bug is already reported** in the issues.
- **Use a clear and descriptive title** for the issue.
- **Describe the exact steps which reproduce the problem** in as many details as possible.
- **Explain which behavior you expected to see and why.**
- **Include screenshots** if the problem is visual.

### Suggesting Enhancements
This section guides you through submitting an enhancement suggestion, including completely new features and minor improvements to existing functionality.

- **Check if the enhancement is already suggested.**
- **Use a clear and descriptive title.**
- **Provide a step-by-step description of the suggested enhancement** in as many details as possible.
- **Explain why this enhancement would be useful** to most users.

### Pull Requests
- **Create a new branch** for your changes.
- **Include screenshots/videos** for UI changes.
- **Ensure the code passes linting and type checks** (`npm run lint`, `npm run typecheck`).
- **Update documentation** if you're changing behavior or adding features.

## Styleguides

### Git Commit Messages
- Use the present tense ("Add feature" not "Added feature").
- Use the imperative mood ("Move cursor to..." not "Moves cursor to...").
- Limit the first line to 72 characters or less.

### JavaScript/TypeScript Styleguide
- Use TypeScript for all new files.
- Follow the existing ESLint configuration.
- Use functional components and hooks for React.

## Questions?
If you have any questions, feel free to open an issue or reach out to the maintainers.
